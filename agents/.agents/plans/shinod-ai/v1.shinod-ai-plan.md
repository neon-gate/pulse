# PURPOSE

Shinod AI provides acoustic fingerprinting, transcription, and AI-based track approval.

Responsibilities:

```
1. Petrified: Generate acoustic fingerprints for uploaded tracks (Chromaprint/fpcalc)
2. Fort Minor: Transcribe audio to text (OpenAI Whisper via AI SDK)
3. Stereo: Combine fingerprint + transcription, run AI reasoning to approve or reject tracks
4. Emit track.approved or track.rejected for downstream transcoding
```

Shinod AI **does NOT**:

```
transcode audio
persist HLS
serve playback
own user profiles
```

Those belong to Mockingbird, Hybrid Storage, and other services.

Fort Minor and Stereo both consume `track.petrified.generated` with **distinct queue groups** to avoid competing consumers (fan-out semantics).

# MODULES

## Petrified

**Role:** Acoustic fingerprinting via Chromaprint/fpcalc.

**Ports:** PetrifiedGeneratorPort, AudioHashPort, IdempotencyPort, PetrifiedEventBusPort

**Consumes:** `track.uploaded`

**Emits:** `track.petrified.generated`, `track.petrified.song.unknown`, `track.petrified.detected`, `track.petrified.failed`, `track.duplicate.detected`

**Adapters:** ChromaprintAdapter, RedisAudioHashAdapter, RedisIdempotencyAdapter

---

## Fort Minor

**Role:** Transcribe audio to text using OpenAI Whisper (AI SDK).

**Ports:** TranscriberPort, IdempotencyPort, FortMinorEventBusPort

**Consumes:** `track.petrified.generated` (queue: `{NATS_QUEUE_GROUP}-fort-minor-petrified`)

**Emits:** `track.fort-minor.started`, `track.fort-minor.completed`, `track.fort-minor.failed`

**Adapters:** AiSdkTranscriberAdapter, RedisIdempotencyAdapter

---

## Stereo

**Role:** Aggregate fingerprint and transcription state; run AI reasoning to approve or reject.

**Ports:** StereoPort, TrackStatePort, IdempotencyPort, StereoEventBusPort

**Consumes:** `track.petrified.generated` (queue: `{NATS_QUEUE_GROUP}-stereo-petrified`), `track.fort-minor.completed`

**Emits:** `track.stereo.started`, `track.approved`, `track.rejected`, `track.stereo.failed`

**Adapters:** AiSdkStereoAdapter, MongoTrackStateAdapter, MongoIdempotencyAdapter

---

# FAN-OUT

Fort Minor and Stereo subscribe to `track.petrified.generated` with **different queue group names**:

- Fort Minor: `{NATS_QUEUE_GROUP}-fort-minor-petrified`
- Stereo: `{NATS_QUEUE_GROUP}-stereo-petrified`

This ensures both modules receive each event (fan-out) rather than competing for the same message (competing consumers).
