# Mockingbird Micro Plan (V1)

## Purpose

Align `Mockingbird` as the transcoding worker that starts from approved tracks and outputs deterministic media artifacts.

## Scope

- Keep ownership of transcoding jobs and transcoded artifact writes.
- Consume only approved tracks with complete source artifact references.
- Emit clear transcoding lifecycle events.

## Current Responsibilities

- Consumes:
  - `track.approved`
- Emits:
  - `track.transcoding.started`
  - `track.transcoding.completed`
  - `track.transcoding.failed`

## Contracts

### HTTP

- `GET /health` (operational endpoint)

### Events

- Consumes: `track.approved`
- Produces: `track.transcoding.started`, `track.transcoding.completed`, `track.transcoding.failed`

## Storage Ownership

- Temporary transcoding workspace (`/tmp/hls`)
- Object storage for transcoded variants
- No ownership of final HLS persistence index

## Alignment Gaps

- `track.approved` payload currently may not provide required object reference.
- Emitted variant keys must match actual storage keys.
- HLS bridge contract must be explicit (`track.hls.generated` or equivalent stage).

## Plan

1. Define strict input contract required for `track.approved`.
2. Normalize variant key contract against physical object paths.
3. Define and align next-stage event handoff with Hybrid Storage pipeline.
4. Add contract-level tests for approval -> transcoding flow.

## Refactor Backlog

1. Enforce strict required fields in `track.approved` consumer contract.
2. Emit deterministic variant artifact metadata (`128`, `320`, keys, bucket, mime).
3. Define downstream handoff event for HLS stage (or contract conversion step).

## Smoke-Test Checklist (Design Only)

- [ ] Invalid `track.approved` payload is rejected with failure event.
- [ ] Valid approved track emits started/completed with artifact refs.
- [ ] Variant keys resolve to physical objects in configured storage.
- [ ] Downstream stage can consume handoff contract without implicit assumptions.

## Acceptance Criteria

- No transcoding starts without valid source artifact contract.
- Emitted variant references are reliable for downstream services.
- Transcoding stage cleanly hands off to next pipeline stage by documented contract.
