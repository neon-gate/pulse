# PURPOSE

Authority provides authentication and session management.

Responsibilities:

```
1. Email/password signup and login
2. Google OAuth signup and login
3. JWT access and refresh tokens
4. Session lifecycle (create, refresh, logout)
5. Publish auth events for downstream services
6. Consume user.profile.created to link profileId
```

Authority **does NOT**:

```
own user profiles
store display names or preferences
serve profile CRUD
```

Those belong to Slim Shady.

# IDENTITY CONTEXT

```
Client
  │
  ▼
Authority (signup/login)
  │
  ▼
authority.user.signed_up
  │
  ▼
Slim Shady (profile creation)
  │
  ▼
user.profile.created
  │
  ▼
Authority (update profileId on User)
```

Authority emits `authority.user.signed_up` when a user registers. Slim Shady creates a profile and emits `user.profile.created`. Authority consumes that event to update the user's profileId via `UserPort.updateProfileId`.

---

# EVENTS EMITTED

```
authority.user.signed_up
authority.user.logged_in
authority.token.refreshed
authority.user.logged_out
```

Example `authority.user.signed_up`:

```json
{
  "userId": "uuid",
  "email": "user@example.com",
  "provider": "password",
  "name": null,
  "occurredAt": "2026-03-15T12:00:00Z"
}
```

---

# EVENTS CONSUMED

```
user.profile.created
```

Handled by `UserProfileCreatedConsumer` to link `profileId` to the user via `UserPort.updateProfileId(userId, profileId)`.

# PORTS

| Port | Responsibility |
|------|----------------|
| UserPort | findByEmail, findById, create, updateProfileId |
| SessionPort | create, findById, update, deleteById |
| GoogleOAuthPort | verifyIdToken(idToken) → GoogleProfile |
| AuthorityEventBusPort | Typed event bus for AuthorityEventMap |

---

# ADAPTERS

| Port | Adapter |
|------|---------|
| UserPort | MongooseUserAdapter |
| SessionPort | SessionCircuitBreakerAdapter → MongooseSessionAdapter |
| GoogleOAuthPort | GoogleOAuthAdapter (google-auth-library) |
| AuthorityEventBusPort | NatsEventBusAdapter (or NoopEventBusAdapter when NATS off) |

# EXPECTED BEHAVIOR

Flow:

```
signup
   │
   ▼
Authority (create User, Session)
   │
   ▼
emit authority.user.signed_up
   │
   ▼
Slim Shady creates profile
   │
   ▼
emit user.profile.created
   │
   ▼
Authority updates User.profileId
```

Login flow:

```
login
   │
   ▼
Authority (validate credentials, create session)
   │
   ▼
emit authority.user.logged_in
   │
   ▼
return accessToken + refreshToken
```
