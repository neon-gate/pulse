# PURPOSE

Slim Shady owns user profiles and preferences.

Responsibilities:

```
1. Consume authority.user.signed_up
2. Create user profile on signup
3. Emit user.profile.created
4. Expose profile CRUD via HTTP
5. Update preferences and onboarding state
```

Slim Shady **does NOT**:

```
authenticate users
manage sessions
issue tokens
```

Those belong to Authority.

# IDENTITY CONTEXT

```
Authority (signup)
  │
  ▼
authority.user.signed_up
  │
  ▼
Slim Shady (UserSignedUpConsumer)
  │
  ▼
CreateUserProfileUseCase
  │
  ▼
emit user.profile.created
  │
  ▼
Authority (link profileId to User)
```

# EVENTS EMITTED

```
user.profile.created
user.profile.updated
user.profile.deleted
```

Example `user.profile.created`:

```json
{

}
```

| Port | Responsibility |
|------|----------------|

# HTTP API

| Method | Path | Description |
|--------|------|-------------|
| GET | /slim-shady/profile | Get current user profile |
| POST | /slim-shady/profile | Create profile (or update) |
| PATCH | /slim-shady/profile | Update profile fields |
| PATCH | /slim-shady/profile/preferences | Update preferences |
| POST | /slim-shady/profile/onboarding/complete | Complete onboarding |
