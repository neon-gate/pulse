## Findings

| Rule | Status | Location |
|------|--------|----------|
| Descriptive Generics Only | FAIL | `redis.adapter.ts:18,22` - `get<T>` and `set<T>` use `T` prefix |
| No Shared Infrastructure | MINOR | `ports/cache.port.ts` duplicates kernel's CachePort; `redis.adapter` imports from `@pack/kernel`. Unused duplicate. |

## Remediation

1. Change `get<T>` to `get<CachedValue>` and `set<T>` to `set<CachedValue>` in `redis.adapter.ts`.
2. Remove `ports/cache.port.ts` (unused duplicate) and ensure all consumers use `@pack/kernel` CachePort; or consolidate if cache package should own the port.
