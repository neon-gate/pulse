## Findings

| Rule | Status | Location |
|------|--------|----------|
| Semantic Suffixes | FAIL | `persist-hls-package.use-case.ts` - should be `persist-hls-package.usecase.ts` (table specifies `.usecase`) |
| Use Cases As Classes | FAIL | `persist-hls-package.use-case.ts:9` - `PersistHLSPackageUseCase` does not extend `UseCase` from `@pack/kernel` |
| No Shared Infrastructure | FAIL | `infra/env/` duplicates `@env/lib`; should use `requireStringEnv`, `requireNumberEnv` from `@env/lib` |
| Entities Are Kernel Entities | FAIL | `hls-package.entity.ts:7` - `HLSPackage` is plain class; if entity, should extend `Entity` or `AggregateRoot` |
| Never Default Environment Variables | MINOR | `infra/env/` - local env helpers; remove and use `@env/lib` |
| Logging | FAIL | `main.ts:12`, `persist-hls-package.use-case.ts:45`, `mock-hls-generator.service.ts:19,30`, `hls-generated.consumer.ts:26,33,41` - uses `console.log`/`console.error` instead of NestJS Logger |
| Ports As Abstract Classes | FAIL | `noop-event-bus.adapter.ts:4` - `implements EventBus<Events>` but `EventBus` is abstract class; should use `extends` |

## Remediation

1. Rename `persist-hls-package.use-case.ts` to `persist-hls-package.usecase.ts`.
2. Refactor `PersistHLSPackageUseCase` to extend `UseCase<[HLSPackage], void>` from `@pack/kernel`.
3. Remove `infra/env/` and replace usages with `@env/lib` (`requireStringEnv`, `requireNumberEnv`, `optionalStringEnv`).
4. Either make `HLSPackage` extend `Entity`/`AggregateRoot` if it is a domain entity, or move to a different layer (e.g. DTO/value object) with appropriate naming.
5. Replace all `console.log`/`console.error` with NestJS `Logger` or structured logging.
6. Change `NoopEventBusAdapter` from `implements EventBus<Events>` to `extends EventBus<Events>`.
