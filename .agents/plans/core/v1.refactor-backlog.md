## P0 - Event Contract Breaks

### 1) Fix Petrified fan-out competing consumers

- Problem: Fort Minor and Stereo share queue-group semantics on `track.petrified.generated`.
- Target: both stages reliably receive each event.
- File targets:
  - `domain/ai/shinod-ai/src/modules/fort-minor/interface/consumers/petrified-generated.consumer.ts`
  - `domain/ai/shinod-ai/src/modules/stereo/interface/consumers/petrified-generated.consumer.ts`
- Acceptance criteria:
  - each uploaded track reaches both transcription and reasoning readiness path
  - no probabilistic stage drop due to queue competition

### 2) Normalize `track.approved` payload for Mockingbird

- Problem: approval event may omit required source object reference.
- Target: strict payload contract consumed by Mockingbird.
- File targets:
  - `domain/ai/shinod-ai/src/modules/stereo/application/use-cases/run-stereo.use-case.ts`
  - `domain/streaming/mockingbird/src/mockingbird/interface/consumers/track-approved.consumer.ts`
  - `domain/streaming/mockingbird/src/mockingbird/domain/events/mockingbird-event.map.ts`
- Acceptance criteria:
  - Mockingbird can always resolve source object for approved tracks
  - contract validation failure paths are explicit and observable
