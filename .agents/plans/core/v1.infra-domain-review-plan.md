# Infra + Domain Review Plan (V1)

## Objective

Define how to review `docker-compose.yml` and `domain/*` boundaries for architecture alignment without executing containers.

## Inputs

- `docker-compose.yml`
- `domain/identity/*`
- `domain/streaming/*`
- `domain/ai/*`
- `domain/realtime/*`
- `.agents/context/ARCHITECTURE.md`
- `.agents/context/EVENTS_PIPELINE.md`

## Review Checklist

1. Service-to-infra dependency map from compose:
   - db/cache/nats/storage dependency correctness
   - volume coupling risks
2. Transport boundary map by service:
   - HTTP vs NATS vs Socket.IO
3. Persistence ownership matrix:
   - Mongo, Redis, object storage, tmp paths
4. Event contract map:
   - subject naming consistency
   - producer/consumer completeness
5. Boundary leak detection:
   - auth/profile separation
   - projection-only constraints in Backstage
   - Shinod internal module coupling risks

## Expected Output

- updated truth matrix (`.agents/context/SERVICE_TRUTH_MATRIX.md`)
- prioritized backlog entries in `.agents/plans/code-audit/v1.refactor-backlog.md`
- smoke-test coverage references in `.agents/plans/code-audit/v1.final-plan.md`
