## End-to-End Validation Strategy

1. Validate static contract consistency first (event names, payload shape, producer/consumer parity).
2. Validate stage-by-stage event propagation using deterministic `trackId` fixtures.
3. Validate negative paths (`upload.failed`, `duplicate`, `rejected`, stage failures).
4. Validate projection and websocket behavior in Backstage from real runtime subjects only.
5. Validate identity loop (`authority.user.signed_up` -> `user.profile.created` -> Authority linkage).

## Smoke-Test Matrix (Design Only)

| Stage | Scenario | Expected Events | Pass Signal |
| --- | --- | --- | --- |
| Identity | Signup user | `authority.user.signed_up`, `user.profile.created` | Authority user linked with `profileId` |
| Identity | Login/logout/refresh | `authority.user.logged_in`, `authority.user.logged_out`, `authority.token.refreshed` | `/authority/me` contract remains stable |
| Ingestion | Upload valid track | `track.upload.received`, `track.upload.validated`, `track.upload.stored`, `track.uploaded` | `track.uploaded` includes canonical artifact refs |
| AI | Fingerprint + transcription + reasoning | `track.petrified.generated`, `track.fort-minor.completed`, `track.approved|track.rejected` | both Fort Minor and Stereo process same track deterministically |
| AI negative | Duplicate or failed processing | `track.duplicate.detected` or `track.*.failed` | terminal state visible in Backstage |
| Transcoding | Approved track transcode | `track.transcoding.started`, `track.transcoding.completed` | output variant refs map to durable objects |
| HLS | Persist generated package | `track.hls.generated`, `track.hls.stored` | manifests/segments stored with canonical keys |
| Realtime | Pipeline projection | `track.*` consumed and broadcast as `pipeline.event` | frontend receives ordered stage updates |
