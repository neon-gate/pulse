# Backstage Micro Plan (V1)

## Purpose

Align `Backstage` as realtime observability/projection service for pipeline events, without becoming a business workflow owner.

## Scope

- Keep ownership of pipeline event projection and websocket broadcast.
- Consume `track.*` events for monitoring and UI visibility.
- Avoid mutating business-domain workflows.

## Current Responsibilities

- Consumes wildcard:
  - `track.>`
- Broadcasts:
  - `pipeline.event` via Socket.IO
- Persists pipeline history/read models

## Contracts

### HTTP

- `GET /pipelines`
- `GET /pipelines/:trackId` (read-model endpoints)

### Events

- Consumes: `track.>`
- Produces: none on NATS (projection service)
- Websocket output: `pipeline.event`

## Storage Ownership

- MongoDB read models and pipeline history projections
- No ownership of source domain aggregates/artifacts

## Alignment Gaps

- Payload rebroadcast policy should prevent leaking internal-only fields.
- `track.ready` and mock-only subjects need clear separation from real flow.
- Read-model lifecycle states should match actual producer events.

## Plan

1. Define allowed payload surface for websocket projection.
2. Split real runtime subjects from mock/dev subjects in docs and contracts.
3. Align pipeline terminal states with actual emitted events.
4. Add observability-focused smoke-test checklist.

## Refactor Backlog

1. Explicitly whitelist fields projected to websocket clients.
2. Separate mock event generator subjects from runtime subjects in config/docs.
3. Align UI pipeline states to real terminal events (`approved`/`rejected`/failed stages).

## Smoke-Test Checklist (Design Only)

- [ ] Backstage consumes real `track.*` events and persists projection timeline.
- [ ] `pipeline.event` payload never includes internal-only sensitive fields.
- [ ] Mock generator can be toggled off without breaking runtime projections.
- [ ] Pipeline read-model reflects event ordering and terminal status consistently.

## Acceptance Criteria

- Backstage remains read/projection-only.
- Realtime payload contracts are explicit and safe.
- Pipeline UI states map to real event lifecycle.
