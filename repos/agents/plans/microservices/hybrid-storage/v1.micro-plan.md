# Hybrid Storage Micro Plan (V1)

## Purpose

Align `Hybrid Storage` as the HLS artifact persistence/delivery storage boundary with clear upstream contract dependencies.

## Scope

- Keep ownership of persisted HLS package layout and object writes.
- Consume finalized HLS package events.
- Emit storage completion events when writes are durable.

## Current Responsibilities

- Consumes:
  - `track.hls.generated`
- Declares:
  - `track.hls.stored`
- Provides health endpoint

## Contracts

### HTTP

- `GET /health` (operational endpoint)

### Events

- Consumes: `track.hls.generated`
- Produces: `track.hls.stored` (modeled; runtime emission must be validated)

## Storage Ownership

- Temporary HLS staging (`/tmp/hls`)
- Durable object storage for manifests and segments
- Ownership of canonical HLS package key layout

## Alignment Gaps

- Real upstream producer for `track.hls.generated` is incomplete in non-mock flow.
- `track.hls.stored` emission path needs explicit contract and consumers.
- Canonical HLS key schema must be pinned and shared.

## Plan

1. Define canonical HLS package payload in `agents/context/EVENTS_PIPELINE.ms`.
2. Define storage key ownership and naming guarantees.
3. Plan completion event emission and downstream consumption expectations.
4. Add persistence contract validation checklist.

## Refactor Backlog

1. Define canonical `track.hls.generated` payload contract (manifest + segments + storage target).
2. Implement/verify `track.hls.stored` emission and declare consumer map.
3. Align mock mode with real mode contract fields to prevent drift.

## Smoke-Test Checklist (Design Only)

- [ ] HLS package event with valid payload stores all expected artifacts.
- [ ] Missing/invalid package fields fail fast and emit failure diagnostics.
- [ ] `track.hls.stored` includes deterministic manifest key and track context.
- [ ] Mock mode and runtime mode expose the same contract shape.

## Acceptance Criteria

- HLS ingest contract is unambiguous.
- Stored artifact paths are deterministic and documented.
- Completion event path is explicitly mapped end-to-end.
