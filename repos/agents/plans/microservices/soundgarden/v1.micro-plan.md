# Soundgarden Micro Plan (V1)

## Purpose

Align `Soundgarden` as the ingestion edge for upload workflow and event handoff to AI stages.

## Scope

- Keep ownership of upload validation and initial artifact storage.
- Emit upload lifecycle events with stable contracts.
- Avoid taking responsibility for AI, transcoding, or playback.

## Current Responsibilities

- Upload HTTP endpoint
- Validation and local/object storage handoff
- Emits:
  - `track.upload.received`
  - `track.upload.validated`
  - `track.upload.stored`
  - `track.uploaded`
  - `track.upload.failed`

## Contracts

### HTTP

- `POST /tracks/upload`

### Events

- Produces: `track.upload.received`, `track.upload.validated`, `track.upload.stored`, `track.uploaded`, `track.upload.failed`
- Consumes: none (edge ingress producer)

## Storage Ownership

- Local temp uploads (`/tmp/uploads`)
- Object storage handoff for source artifact
- No ownership of AI/transcoding derivative artifacts

## Alignment Gaps

- Bucket/key conventions drift (`tracks` vs `uploads`) must be standardized.
- `track.uploaded` must always include downstream-required artifact references.
- Deprecated payload keys should have a retirement strategy.

## Plan

1. Define canonical upload artifact contract in `agents/context/EVENTS_PIPELINE.ms`.
2. Normalize object storage key schema used by downstream AI modules.
3. Plan phased removal of deprecated payload aliases.
4. Add ingestion contract smoke checks.

## Refactor Backlog

1. Normalize upload artifact fields and deprecate compatibility aliases in phased rollout.
2. Pin bucket/key naming convention for source audio objects.
3. Add explicit event-contract tests for `track.uploaded` minimum fields.

## Smoke-Test Checklist (Design Only)

- [ ] Successful upload emits full lifecycle events in order.
- [ ] Failed upload emits `track.upload.failed` with track/request context.
- [ ] `track.uploaded` payload always includes downstream artifact reference.
- [ ] Storage contract remains stable across local and object-storage modes.

## Acceptance Criteria

- Upload events are complete and downstream-safe.
- Storage artifact fields are consistent across producers/consumers.
- Service remains a pure ingress boundary.
